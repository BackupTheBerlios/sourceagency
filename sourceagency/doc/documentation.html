2. Introduction

SourceAgency is a highly configurable, well documented Open Source project announcement and retrieval system entirely writen in PHP3 and database independent. It includes user authentication and autherization system (devel/sponsor/editor/admin), sessions with and without cookies, high configurability, multilangual support, ease of administration, RDF-type document backend, advanced statistics, announcing mailing lists, project indexing by sorts and many other useful features.

SourceAgency depends on the PHPLib library (version 7.2 or later). Only if you want to have diary and weekly mailing lists with the announcements, you should also have Mailman installed in your box.

You can see a fully working example of the SourceAgency system at BerliOS SourceAgency by visiting http://sourceagency.berlios.de. A close look at it will show you what you can do with SourceAgency. BerliOS SourceAgency has at this moment more than ... projects inserted and has been the main reason why we have made this software.

BerliOS SourceAgency is part of the BerliOS project at GMD FOKUS. Please, have a look at http://www.berlios.de for further information.

You can download the latest version of SourceAgency at: http://developer.berlios.de/projects/sourceagency

SourceAgency Features:

  • Different type of users (nonauthorized users, developers, sponsors, editors and administrators) with different functions
  • Session management with and without cookies
  • Database independence
  • Advanced configurability from a single file
  • Simple, intuitive use of the system
  • Documentation for further development and/or adjustment
  • Comments on projects
  • Through-the-web reviewing and project administration for editors
  • Through-the-web administration of projects, comments and licenses
  • system FAQ and through-the-web administration of it
  • Dynamic order of projects by date (default), interest, urgency or by alphabetical order
  • "true" project counter for hits, etc.
  • Multilingual support
  • Dynamic permission configuration
  • XML Backend (RDF-document format)
  • Daily and Weekly automatic Newsletters
  • EMail advice for editors when projects are inserted or updated
  • EMail advice for administrators when new users register
  • Web browser independence
  • Cache avoidance

 

3. Installation

Important note about security:

  • SourceAgency's main directory and all its subdirectories are not protected in any way by us! Please make sure that you can put propperly .htaccess files in these directories (especially the include subdirectory). If someone is able to read the local.inc file in the include directory, operator (user name) and password for your database could be known and misused. For further information about .htaccess look at the Apache's documentation.

3.1 Fast installation (for MySQL)

You need to have a webserver (we have tested the system with Apache), PHP3, PHPLib and MySQL installed in your system. Mailman is only needed if you want to use mailing lists.

  • user$ mysql -h host -u database_superuser -p
    (insert superuser_password)
  • mysql> USE mysql;
  • mysql> INSERT INTO user VALUES ( 'host', 'operator', PASSWORD('operator_password'), 'Y', 'Y', 'Y', 'Y', 'Y', 'Y', 'Y', 'N', 'N', 'N', 'N', 'N', 'N', 'N');
  • mysql> FLUSH PRIVILEGES;
  • mysql> CREATE DATABASE sourceagency;
  • mysql> exit;
  • user$ tar -zxvf sourceagency-0.3.tar.gz
  • user$ mysql -h host -u operator -p < sourceagency/sql/mysql.sql
    (insert the operator_password)
  • user$ vi sourceagency/include/local.inc
    (insert the correct database parameters; please, keep sourceagency as the database name) 
       var $Host     = "host";
   var $Database = "sourceagency";
   var $User     = "operator";
   var $Password = "password";

Ok, that's it. You've finished installing and configuring SourceAgency. Start your browser and visit http://yourserver/sourceagency/index.php3. As you'll see, SourceAgency comes with some inserted example projects and licenses. You can delete or modify them by entering as an administrator. To do so, login as the default administrator with admin permissions. At the prompt use the following ID to login (case sensitive): 

        Username: admin
	Password: admin

Very important: Be sure of inmediately changing the login and password by clicking on the User Admin link. You'll notice this is quite easy to do ;-)

If things do not work, maybe you'll have to read the whole installation chapter :-(.

3.2. Before starting

You've got to look if your web server is correctly configurated and running. That means that:

  • The web server is capable of recognising and executing PHP3 scripts in user directories
  • PHP is compliled to work with the database your are going to work with.

Please, attend to the respective manuals and documentation for a propper installation.

3.2.1. About PHPLib

For a correct functioning of SourceAgency, PHPLib has to be installed. PHPLib is an usefull library which permits SourceAgency to have sessions, authentication and database independence. PHPLib is published under the terms of the GNU Lesser General Public License (LGPL). For more information about PHPLib follow this link: http://phplib.netuse.de/.

SourceAgency has been developed with the 7.2c version of PHPLib, which can be downlodaded from http://phplib.netuse.de/download/phplib-7.2c.tar.gz. As SourceAgency strictly uses the API given by PHPLib, future versions of PHPLib should also work without problem.

Please read carefully the documentation provided for PHPLib for a successfull installation. The installation of PHPLib is sufficient; you don't need to know more about PHPLib and its functioning for using the SourceAgency system.

3.2.2. About the Database

The next step is to install and configure the database system. SourceAgency has been tested in BerliOS SourceAgency with MySQL, so that the following explanations are to be followed with that RDBMS. For other systems, the philosophy will be the same, but you will have to particularize the instructions.

You can download MySQL from: http://www.mysql.com/downloads.

1. Install and configure the RDBMS

2. Create the user SourceAgency needs:

  • Login as MySQL's superuser (host should be localhost in the default case)
    user$ mysql -h host -u database_superuser -p
    (insert superuser_password)
  • Choose the mysql database
    (that's where the database users and their privileges are)
    mysql> USE mysql;
  • Insert an operator user for SourceAgency, its password and its privileges
    mysql> INSERT INTO user VALUES ( 'host', 'operator', PASSWORD('operator_password'), 'Y', 'Y', 'Y', 'Y', 'Y', 'Y', 'Y', 'N', 'N', 'N', 'N', 'N', 'N', 'N');
  • "Commit" the changes
    mysql> FLUSH PRIVILEGES;

3. Create the data base:

  • Create the data base (with sourceagency as its name; see note below)
    mysql> CREATE DATABASE sourceagency;
  • Exit MySQL
    mysql> exit;

Note: It is important for a simple configuration of the system to name this database "sourceagency". A different database name would imply editing and modifying many of the default configuration parameters in the local.inc file, which is not recommended.

3.3. Obtaining SourceAgency

You can get the latest version of SourceAgency at: http://developer.berlios.de/projects/sourceagency

3.4. File extraction

Due to security reasons you should not install or execute SourceAgency as root. It's not even necessary.

The installation process is as following:

  1. choose the parent directory where you want SourceAgency to be and cd into it
  2. verify that the user public_html exists and is accesible from the web
  3. uncompress SourceAgency in the parent directory
    user$ tar -zvxf sourceagency-0.3.tar.gz
  4. List the content of the directories. You should have a directory sourceagency with five subdirectories. Read the point "Files listed and shortly explained" for further information about the files.
    user$ ls -R
  5. create the database structures in the database
    user$ mysql -h host -u operator -p < sourceagency/sql/mysql.sql
    (insert the operator_password)

3.5. Database configuration

The database configuration has to be done in local.inc; both are in the include directory. Please, at this point read another time the security note given at the beginning of point 3.

  • user$ vi sourceagency/include/local.inc
    (insert the correct database parameters; please, keep sourceagency as the database name) 
       var $Host     = "host";
   var $Database = "sourceagency";
   var $User     = "operator";
   var $Password = "password";

Congratulations, you've finished installing and configuring SourceAgency. Start your browser and visit http://yourserver/sourceagency/index.php3. As you'll see, SourceAgency comes with some inserted example projects and licenses. You can delete or modify them by entering as an administrator. To do so, login as the default administrator with devel,sponsor,editor,admin permissions. At the prompt use the following ID to login (case sensitive): 

        Username: admin
	Password: admin

Vert important: Be sure of inmediately changing the login and password by clicking on the link User Admin. You'll notice this is quite easy to do ;-)

5. Main Functionality

Note: In the following, we are going to talk about the default permission configuration of the system (which is also the recommended setting). The permission configuration has been implemented to be dinamically, so that the site administrator can easily modify the permissions needed at his will from the include/config.inc file.

Authorisation is based on registration. The SourceAgency system has two sets of users: unregistered users and registered users. Between registered users we have different permission levels:

  • developer
  • sponsor
  • editor
  • admin

Permissions are independent and do not have to be unique for a certain registered user. That means that a registered user with only administrator permissions is not allowed to perform actions that are restricted to registered users with editor or user permissions, but a single registered user can have devel, sponsor, editor and admin permissions.

Each level of authorisation comprises certain actions that are listed below. Again, we want the reader to notice, that this is the default configuration. The administrator can change it at his will at the include/config.inc file:

Unregistered users can:

  • Retrieve OpenSource projects
  • Retrieve already finished OpenSource projects
  • View the comments on projets
  • Read the FAQ
  • Choose language
  • Login
  • Obtain the forgotten password
  • Become authorised user by entering a form
  • Read the XML Backend
  • Search for projects
  • See list of developers and sponsors
  • See developers' and sponsors' information

Registered users with developer permission can:

  • Do everything what unregistered user can do
  • Insert and update projects
  • Write comments on projects
  • Change their user registration data
  • Rate the sponsor when they finish a project
  • See further information of projects they are related in
  • Have a personal page
  • Make their rating public/private

Registered users with sponsor permission can:

  • Do everything what unregistered user can do
  • Insert and update projects
  • Write comments on projects
  • Change their user registration data
  • Rate the developers when they finish a project
  • See further information of projects they are related in
  • Have a personal/institutional page
  • Make their rating public/private

Editors (editor permission) are able to:

  • Do everything what unregistered users can do
  • Review pending projects (inserted or updated by registered users)
  • Delete and undelete projects

Administrators (admin permission) are able to:

  • Do everything what unregistered users can do
  • Administrate users and their permissions
  • Administrate (insert, rename, change URL and delete) licenses
  • Administrate (modify and delete) comments on projects

As explained before, SourceAgency's registered users can have several roles. So it is possible (and recommendable) that editors (editor) have also devel and sponsor permissions and that administrators (admin) have also developer, sponsor and editor permissions. This becomes even more important if you personalize the permissions of your system in the include/config.inc file as you can only choose one permission type at a time (whether user (developer or sponsor), editor, admin or all) for every type of functionality.

 

5. International support

SourceAgency can be easily translated into different languages. If you see that SourceAgency does not have support in your language, you're gladly invited to help us with the internationalization of SourceAgency by sending us your translation.

You don't need to have any computer or programming experience to do a translation. Keep on reading and you'll find out how easy it is.

5.1. Main outputs

Download the lang-translate.inc file (it also comes in SourceAgency's 0.3 tarball) . If you edit it, you'll find lines like this: 

     case "Home": $tmp = ""; break;

We will explain it briefly: after the case you will see the English text to translate writen in quotes (in our example, the English text is "Home"). Then you'll find a sort of equation. The content of your translation from English into your language should be placed in between these second quotes. For example, in the case you were making a translation into German, this would be the result for this line: 

     case "Home": $tmp = "Heim"; break;

Ok, now that you're an expert, you'll notice that "Home" is translated into German as "Heim" ;-). The procedure just explained should be repeated with all the lines in this file.

Once you're finished, save it as YourLanguage-lang.inc and please send it to the authors. We will include it in the next releases so that everybody can benefit of your work.
